BFC及其应用

2017-07-10

何为BFC

BFC（Block Formating Context 块级格式上下文）是CSS2.1规范中提出的一个概念，他是页面中独立的一块渲染区域。决定了这个区域内部元素的渲染以及与区域外部其他元素的渲染关系。它内部元素的渲染不会影响外部元素。

BFC的布局规则

内部的BOX会一个接一个的放置
box垂直方向的距离由margin决定，属于同一个BFC的相邻box会发生margin折叠
每个元素margin box的左边，与包含块的border box的左边相接触。即便存在浮动也是如此。
一个独立的容器，里面的元素不会影响外部。
容器的高度包含浮动元素的高度。
BFC的触发

根元素
float属性不为none
position为absolute或者fixed
display为inline-block，table-cell，table-caption，flex，inline-flex
BFC的应用

自适应两栏布局





.wapper{
width:300px;
background-color:red;
position:absolute;
}
.left{
float:left;
width:100px;
height:200px;
background-color:black;
}
.right{
height:200px;
background-color:green;
overflow:hidden;
}
根据BFC规则第三条：

每个元素的margin box的左边与包含块的左边相接触，浮动也如此.
另外.right由于有overflow:hidden所以新生成了BFC。根据第四条:

BFC的区域不会和float box重叠
这种以BFC特性的两栏布局虽然不需要和常见的流体布局一样关心固定栏的宽度，但是由于float的缘故，使得固定栏失去了流体特性，从而无法达到自动填满容器的布局。这里要注意float元素的破坏性和包裹性

清除内部浮动
当设置一个浮动元素的父元素的CSS属性以来触发一个新的BFC的时候，内部元素因为浮动产生的高度塌陷问题就可以解决。这是因为BFC计算高度的时候浮动元素也会计算在内。
防止垂直margin重叠
与上一条同理。如果两个元素同属一个BFC那么在布局的时候相邻元素的margin会发生折叠，这个时候我们需要通过设置CSS属性来生成一个新的BFC避免margin折叠。常见的就是设置这个元素overflow:hidden.
